Design Philosophy

3 Mantras哲学

01
⚡

Instinctive

Design should be instinctive for the user without relying on instructions. If you need to explain it, redesign it. Clarity is the highest form of craft.

02
🔄

Iterative

There is no finish line to design. With each iteration, an opportunity emerges to get it better for the user. Ship, learn, improve — endlessly.

03
🤝

Open

Be receptive to feedback from others, creating a collaborative and inclusive approach to design. The best ideas come from everywhere and everyone.

Vercel Deployment Guide展開

  1. 01

    Create a GitHub Repository

    Push your portfolio files to a new GitHub repo. Vercel deploys directly from Git. Run git init → git add . → git commit -m "init" → push to GitHub.

  2. 02

    Connect to Vercel

    Go to vercel.com → "Add New Project" → Import your GitHub repo. Vercel auto-detects the framework. For a static HTML site, no build settings needed — just deploy.

  3. 03

    Link Your Custom Domain

    In Vercel Dashboard → Project → Settings → Domains → Add your domain (e.g. vijitbhati.com). Copy the provided nameservers or CNAME record to your domain registrar (GoDaddy, Namecheap, etc.).

  4. 04

    Enable HTTPS (Auto)

    Vercel provisions a free SSL certificate via Let's Encrypt automatically. DNS propagation takes 10–48 hours. Your site will be live at https://yourdomain.com with zero config.

  5. 05

    Continuous Deploy

    Every push to your main branch auto-triggers a new deployment. Preview URLs are created for every PR. Rollback to any prior deploy in one click from the dashboard.

🎨 Figma MCP Integration

To edit this design in Figma via MCP: Enable the Figma MCP server in Claude's tool settings. You can then ask Claude to push component specs, export tokens, or sync design variables directly to a Figma file. The recommended workflow is: design tweaks in Figma Dev Mode → export as CSS variables → paste into this file's :root block. For full design-to-code sync, use Tokens Studio plugin in Figma paired with a GitHub Action that commits updated token files on publish.
